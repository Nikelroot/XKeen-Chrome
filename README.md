# XKeen Chrome Extension

Chrome Extension (Manifest V3) для быстрого добавления/удаления домена текущей вкладки в роутинг Xray через XKeen-UI API.

Расширение работает с файлом:

- `/opt/etc/xray/configs/05_routing.json`

и не требует ручного редактирования JSON в панели.

## Что умеет

- Показывает домен активной вкладки (с нормализацией).
- Автоматически проверяет при открытии popup статус сервиса XKeen.
- Автоматически проверяет при открытии popup наличие правил для текущего домена.
- Загружает доступные теги из конфигов (`GET /api/configs`).
- `inboundTag` выбирается как `multiselect`.
- `outboundTag` выбирается как обычный `select`.
- Добавляет домен в `05_routing.json`.
- Для каждого выбранного `inboundTag` создается/обновляется отдельное правило.
- Новые правила вставляются перед правилом с `"comment": "end field"`.
- Если маркер не найден, правило добавляется в конец `routing.rules`.
- Удаляет домен из всех подходящих правил (по выбранным `inboundTag` + `outboundTag`).
- Перезапускает сервис кнопкой `Перезапустить сервис`.
- Автоматически сохраняет настройки в `chrome.storage.sync` (кнопка сохранения не нужна).

## Важные детали логики

- Формат домена: `domain:example.com`.
- `www.` удаляется.
- IP-адреса и пустые значения не добавляются.
- Для одного domain может существовать несколько правил (по разным inbound).
- Проверка `Уже есть` считается успешной только если домен найден для всех выбранных `inboundTag`.

## Обязательные требования

- Роутер с установленными XKeen и XKeen-UI.
- XKeen-UI: [https://github.com/zxc-rv/XKeen-UI](https://github.com/zxc-rv/XKeen-UI)
- Доступный XKeen-UI API (по умолчанию `http://192.168.28.1:1000`).
- Успешный вход в XKeen-UI (активная cookie-сессия в браузере).
- Доступ к файлу роутинга через API: `/opt/etc/xray/configs/05_routing.json` (или ваш путь в настройке `Файл роутинга (.json)`).
- В JSON должен быть массив `routing.rules` (если его нет, расширение создаст только при добавлении, но для корректной проверки он должен читаться).

## Рекомендуется

- В `05_routing.json` иметь маркер конца field-блока:

```json
{
  "comment": "end field"
}
```

## Установка (Developer mode)

1. Откройте `chrome://extensions`.
2. Включите `Developer mode`.
3. Нажмите `Load unpacked`.
4. Выберите папку `xkeen-chrome-extension`.

## Настройка

В popup укажите:

- `Адрес XKeen-UI API` (пример: `http://192.168.28.1:1000`)
- `Файл роутинга (.json)` (по умолчанию `/opt/etc/xray/configs/05_routing.json`)
- `inboundTag` (можно выбрать несколько)
- `outboundTag` (один)

Настройки сохраняются автоматически.

## Использование

1. Откройте сайт в активной вкладке Chrome.
2. Откройте popup расширения.
3. Выберите нужные `inboundTag` и `outboundTag`.
4. Нажмите `Добавить домен`, чтобы добавить правило(а).
5. Нажмите `Удалить домен`, чтобы убрать домен из соответствующих правил.
6. Нажмите `Проверить статус`, чтобы проверить наличие правил.
7. Нажмите `Перезапустить сервис`, чтобы отправить команду рестарта.

## Статусы в UI

- `Добавлено`
- `Уже есть`
- `Удалено`
- `Не найдено`
- `Ошибка авторизации`
- `Ошибка подключения`

При ошибке авторизации:

`Открой XKeen-UI и войди в панель, потом повтори действие`.

## API

Основные запросы идут с `credentials: "include"`:

- `GET /api/configs` — чтение конфигов и тегов.
- `PUT /api/configs` — запись обновленного `05_routing.json`.

Для рестарта сервиса используется последовательность попыток:

- `POST /api/control/restart`
- `POST /api/control` с `{ "action": "restart" }`
- `PUT /api/control` с `{ "action": "restart" }`
- `POST /api/service/restart`
- `POST /api/restart`
